# v0.3.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish Supa.cc 0.3.0 from the validated source commit to GitHub, PyPI, and the project Homebrew tap.

**Architecture:** Use the existing GitHub Release event as the single trigger for building and publishing Python artifacts through PyPI Trusted Publishing. Treat the candidate commit, the immutable tag, PyPI verification, the Homebrew formula update, and final documentation as sequential gates; do not cross a gate after a failed check.

**Tech Stack:** Git, GitHub CLI and Actions, Python 3.9+, pytest, hatchling/build, pip-audit, pipx, PyPI OIDC Trusted Publishing, Homebrew Ruby formula

## Global Constraints

- Release version is exactly `0.3.0`; the Git tag and GitHub Release are exactly `v0.3.0`.
- PyPI publication must use `.github/workflows/release.yml`, Trusted Publishing, and GitHub environment `pypi`; do not create or use a PyPI API token.
- Preserve native credential backends: Keychain on macOS, Secret Service on Linux, and `WinVaultKeyring` on Windows; never introduce plaintext fallback or `keyrings.alt`.
- Do not publish credentials, private paths, personal email addresses, environment dumps, caches, virtual environments, or local artifacts.
- Keep `Formula/supa-cc.rb` at `v0.2.0` until PyPI 0.3.0 has been published and verified.
- Do not attach locally built Python distributions to the GitHub Release.
- Push directly to `main` using the authorized administrative bypass; never force-push.
- Stop before the next irreversible action on any version, artifact, permission, credential, checksum, CI, or repository inconsistency.
- Homebrew publication requires a macOS host with Homebrew for `brew audit`, source installation, and `brew test`; do not publish an unverified formula.
- Do not create Debian, AUR, or RPM assets.

## File Map

- `.github/workflows/ci.yml`: cross-platform candidate validation.
- `.github/workflows/release.yml`: release-event build, Trusted Publishing, and PyPI installation verification.
- `pyproject.toml`: canonical package metadata and version.
- `supa_cc/__init__.py`: runtime version.
- `scripts/inspect_artifacts.py`: wheel and sdist safety inspection.
- `tests/test_publication_assets.py`: assertions for release and distribution configuration.
- `CHANGELOG.md`: release notes, release date, and comparison URL.
- `README.md`: public channel availability.
- `docs/installation.md`: canonical installation availability and commands.
- `docs/release.md`: canonical release runbook.
- `Formula/supa-cc.rb`: Homebrew source URL, checksum, dependencies, resources, and smoke test.

- [ ] **Step 3: Update version URL and checksum in the tap checkout**

```bash
text = File.read(path)
text.sub!(%r{url "https://github.com/dgabreuu/supa\.cc/archive/refs/tags/v0\.2\.0\.tar\.gz"},
          'url "https://github.com/dgabreuu/supa.cc/archive/refs/tags/v0.3.0.tar.gz"')
text.sub!(/sha256 "61ffeb04b5c157f71a5e14fdd7a740757d9ac77655c39476feb04ea02ceaf054"/,
          "sha256 \"#{ENV.fetch("ARCHIVE_SHA")}\"")
File.write(path, text)
RUBY
```

```bash

git -C "$tap_repo" diff -- Formula/supa-cc.rb
```
